Site Report

1. Design Choices

The design choices for this project focused on creating a minimalistic, clean layout with a consistent look across all pages. Key choices included:

  • Color Palette: I chose a grayscale base with accent colors (e.g., orange for hover effects) to create a sophisticated, modern aesthetic. This makes the content stand out without overwhelming the user.
  • Font: The ‘Poppins’ font was selected for its clean, rounded appearance, which adds to the site’s approachable feel.
  • Navigation Bar: The navbar is sticky for easy access as the user scrolls, and includes a hamburger menu on mobile, made using CSS-only techniques.
  • Hero Section Animation: Subtle animations like fade-ins and slide-ups enhance the user experience by adding a dynamic feel without distracting from the content.
  • Responsive Grid Layouts: Used CSS grid for the project page, ensuring the layout adapts well to different screen sizes.

2. Development Challenges

During development, several challenges arose that required thoughtful solutions:

  • Responsive Layout: Initially, using fixed pixel values caused overflow on smaller screens. Switching to a flexible grid with `minmax` allowed the layout to adapt automatically, creating a smoother experience on mobile.
  • Mobile Navigation Menu: Implementing a hamburger menu without JavaScript required experimenting with a hidden checkbox and CSS. Through trial and error, I achieved a clean, functional menu for mobile views.
  • Hover Effects: Balancing subtle yet noticeable hover effects across all elements involved testing different box shadows and transitions to find the right balance. This allowed the design to feel interactive without being too flashy.
  • Form Validation: Implementing form validation in the contact page without using JavaScript initially limited user feedback. I opted to use required fields in HTML as a temporary solution, planning to expand with JavaScript validation in the future.

3. Future Improvements

Several potential enhancements could further improve this website:

  • Dark Mode Toggle: Adding a dark mode feature would improve the user experience, especially for those browsing at night. This could be achieved by adding a CSS variable switcher and a JavaScript toggle.
  • Enhanced Project Descriptions: Expanding each project’s description with a “read more” link or modal would provide more in-depth details without cluttering the page layout.
  • JavaScript Form Validation: Adding JavaScript-based validation for the contact form to improve user feedback and prevent incomplete submissions.

3. Validation Images

  • Validation Reports: HTML and CSS validation screenshots, showing that the code meets W3C standards, reinforcing the project’s quality and adherence to best practices.

6. Conclusion

Building this portfolio site was a valuable learning experience, enabling me to practice responsive design, CSS animations, and creating a clean, professional layout. Through the challenges faced, I developed a greater understanding of front-end design and CSS techniques, and I’m excited to continue enhancing this site with future improvements. This project serves as a foundation for showcasing my skills and growth as a developer.
